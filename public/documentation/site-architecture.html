1. Project Overview

What is ICJIA InfoNet?

InfoNet is a web-based data collection and reporting system used by victim service providers in Illinois. The system is nationally recognized for facilitating standardized data collection and reporting at the statewide level.

Technology Stack

This project uses modern web technologies to deliver a fast, accessible, and maintainable application:

Technology Version Purpose
Nuxt 4 4.1.0 Vue.js framework for building the application
Vue.js 3.x Progressive JavaScript framework for UI
Vuetify 3.7.1 Material Design component library
@nuxt/content 2.13.4 File-based CMS for Markdown content
Strapi N/A (External) Headless CMS for content management
Chart.js 4.4.4 Data visualization library
Fuse.js 7.0.0 Fuzzy search functionality
Axios 1.7.7 HTTP client for API requests

What is JAMstack?

JAMstack Explained (for Beginners)

JAMstack stands for JavaScript, APIs, and Markup. It's a modern web development architecture that delivers better performance, security, and scalability.

How it works:

  • JavaScript: Handles dynamic functionality in the browser
  • APIs: Server-side operations are abstracted into reusable APIs (like Strapi)
  • Markup: Pre-built HTML files served from a CDN (Content Delivery Network)

Key Benefit: Instead of generating pages on-demand when users visit (traditional server-side rendering), JAMstack pre-builds all pages during deployment. This means pages load instantly because they're just static HTML files!

Architecture Overview

Strapi CMS (Content Management)
↓
Build Scripts Fetch Content via GraphQL API
↓
Generate Markdown Files & JSON Data
↓
Nuxt Processes Content with @nuxt/content
↓
Static Site Generation (SSG)
↓
Deploy to Netlify CDN
Why This Architecture?
  • Performance: Pre-built pages load instantly
  • Security: No database or server to hack
  • Scalability: CDN handles millions of requests
  • Developer Experience: Content editors use Strapi, developers use Git

How Nuxt Routing Works

File-Based Routing (for Beginners)

Nuxt automatically creates routes based on files in the app/pages/ directory. You don't need to manually configure routes!

Examples:

  • app/pages/index.vue → / (home page)
  • app/pages/about.vue → /about
  • app/pages/news/index.vue → /news
  • app/pages/news/[slug].vue → /news/any-slug (dynamic route)

Component Organization

Components are organized by purpose in the app/components/ directory:

  • app/components/content/ - Content-specific components (tabs, cards, etc.)
  • app/components/layout/ - Layout components (header, footer, sidebar)
  • app/components/ui/ - Reusable UI components (buttons, modals, etc.)
Naming Convention: Nuxt auto-imports components based on their file path. A component at app/components/content/TabsAccessible.vue can be used as <ContentTabsAccessible /> in any template.

The package.json file is the heart of any Node.js project. It defines:

  • Dependencies: Libraries your project needs to run
  • DevDependencies: Tools needed only during development
  • Scripts: Commands you can run (like yarn dev)

Installing Dependencies

This project uses Yarn as the package manager. Here's how to work with dependencies: 

# Install all dependencies (run this after cloning the repo)
yarn install

yarn add package-name

# Add a development dependency
yarn add -D package-name

# Remove a dependency

4. Strapi Integration & Content Management

What is Strapi?

Strapi Explained (for Beginners)

Strapi is a headless CMS (Content Management System). "Headless" means it only manages content—it doesn't care how you display it.

Traditional CMS (like WordPress):

  • Manages content AND displays it
  • Tightly coupled frontend and backend
  • Limited flexibility

Headless CMS (like Strapi):

  • Only manages content
  • Provides content via API
  • You build the frontend however you want (Nuxt, React, mobile app, etc.)

How Content is Structured in Strapi

Strapi organizes content into Content Types. For InfoNet, we have:

Content Type Purpose Example Fields
Pages Static pages (About, Contact, etc.) title, slug, body, section, summary
Posts (News) News articles and updates title, slug, body, dateOverride, category
FAQs Frequently asked questions question, answer, agency, slug
Tabs Tabbed content sections title, slug, body, sectionID, ranking
Meetings Meeting information title, start, end, body, attachments

Content Areas: DV, SA, and CAC

InfoNet serves three main victim service areas:

  • DV: Domestic Violence services
  • SA: Sexual Assault services
  • CAC: Children's Advocacy Center services

Content in Strapi is tagged with these categories, allowing the site to filter and display relevant information for each service area.

API Endpoints

Strapi provides a GraphQL API for fetching content. The main endpoint is: 

https://infonet.icjia-api.cloud/graphql

Additional APIs used by the project:

  • https://researchhub.icjia-api.cloud/graphql - Research Hub articles
  • https://agency.icjia-api.cloud/graphql - Publications from the agency

Example GraphQL Query

Here's how we fetch pages from Strapi:

query {
  pages {
    data {
      id
      attributes {
        title
        slug
        body
        section
        summary
        createdAt
        updatedAt
      }
    }
  }
}
No Authentication Required: The Strapi APIs used by this project are public and don't require authentication. The build scripts can fetch content without API keys.

How Markdown Files Are Generated

Let's look at how createMarkdownPages.js works:

// 1. Fetch data from Strapi GraphQL API
axios.post('https://infonet.icjia-api.cloud/graphql', { query })
  .then((res) => {
    const pages = res.data.data.pages.data;

    // 2. Process each page
    pages.forEach((page) => {
      // 3. Format as Markdown with YAML frontmatter
      const content = formatMarkdown(page.attributes);

      // 4. Write to file system
      fs.writeFileSync(`content/${slug}.md`, content);
    });

    // 5. Also save as JSON for API routes
    jsonfile.writeFileSync('public/pages.json', pages);
  });

Example Generated Markdown File

---
title: About InfoNet
slug: about
section: root
summary: Learn about the InfoNet system
createdAt: 2024-01-15T10:00:00.000Z
---

This is the page body content from Strapi.
It can include **Markdown** formatting.

How @nuxt/content Processes Markdown

@nuxt/content Explained (for Beginners)

@nuxt/content is a Nuxt module that turns Markdown files into a queryable database. It reads files from the content/ directory and makes them available via a query API.

Example usage in a Vue component:

// Fetch a single page
const { data } = await useAsyncData('about', () =>
  queryContent('/about').findOne()
);

// Fetch all news posts, sorted by date
const { data: posts } = await useAsyncData('news', () =>
  queryContent('/news').sort({ date: -1 }).find()
);

Build vs. Runtime

Understanding when things happen is crucial:

Build Time Runtime (Browser)
Creator scripts run User visits the site
API calls to Strapi No API calls (everything is pre-built)
Markdown files generated HTML files served from CDN
Static pages created Pages load instantly
Key Insight: All content fetching happens at build time, not runtime. This means the site is blazing fast because there are no database queries or API calls when users visit.

6. Development Workflow

Setting Up the Project Locally

Follow these steps to get the project running on your machine:

1. Clone the Repository

git clone https://github.com/ICJIA/icjia-infonet-nuxt4.git
cd icjia-infonet-nuxt4

2. Set Up Environment Variables

# Copy the sample environment file
cp .env.sample .env

# Edit .env with your preferred editor
# The default values should work for most development scenarios

Key environment variables:

NUXT_PUBLIC_API_BASE_URL=https://infonet.icjia-api.cloud
NUXT_PUBLIC_BASE_URL=https://infonet.icjia.dev
NUXT_MANUAL_ROUTES=["/search", "/translate", "/faqs", "/news","/data-and-publications"]

3. Install Node.js (via nvm)

# Install nvm if you don't have it
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.35.3/install.sh | bash

# Use the correct Node.js version (defined in .nvmrc)
nvm use

4. Install Dependencies

yarn install

Running the Development Server

# Start development server on http://localhost:8000
yarn dev

# This command does three things:
# 1. Kills any process on port 8000
# 2. Clears the cache
# 3. Runs creator scripts to fetch latest content
# 4. Starts Nuxt dev server with hot reload
Hot Reload: When you save changes to Vue files, the browser automatically updates without a full page refresh. This makes development much faster!

Triggering Content Updates from Strapi

When content is updated in Strapi, you need to re-run the creator scripts: 

# Fetch latest content from remote Strapi
yarn scripts

# OR fetch from local data (no API calls)
yarn scripts:local
Important: The dev server (yarn dev) automatically runs yarn scripts on startup, so you'll always have the latest content when you start developing.

Common Development Commands

Command Purpose
yarn dev Start development server with hot reload
yarn build Build for production (runs scripts + nuxt build)
yarn generate Generate static site (for deployment)
yarn generate:serve Generate and serve locally on port 3000
yarn scripts Run all creator scripts (fetch from Strapi)
yarn scripts:local Run scripts with local data (no API calls)
yarn clear:cache Clear Nuxt cache (.nuxt and .output)
yarn audit:a11y Run accessibility audit on all routes

Previewing Changes Before Deployment

To see exactly what will be deployed to production:

# Generate static site and serve it locally
yarn generate:serve

# This will:
# 1. Clear cache
# 2. Run creator scripts
# 3. Generate static HTML files
# 4. Serve on http://localhost:3000
Best Practice: Always test with yarn generate:serve before pushing to production. This catches issues that might not appear in dev mode.

7. Deployment

How Netlify Deployment Works

The site is hosted on Netlify, a platform optimized for JAMstack sites. Deployment is fully automated through continuous deployment.

Developer Pushes to GitHub (main branch)
↓
Netlify Detects Changes
↓
Netlify Runs Build Command
↓
Static Files Generated
↓
Deploy to Netlify CDN
↓
Site Live at infonet.icjia.illinois.gov

Netlify Configuration

Deployment settings are defined in netlify.toml:

# Build settings (configured in Netlify dashboard)
Build command: yarn generate
Publish directory: dist
Production branch: main

Build Triggers

Deployment is triggered by:

  • Git Push: Any push to the main branch
  • Manual Deploy: Via Netlify dashboard
  • Webhook: Can be triggered by Strapi when content changes (if configured)

Environment Variables in Netlify

Environment variables are configured in the Netlify dashboard under Site settings → Build & deploy → Environment variables

Important: Make sure all environment variables from .env are also set in Netlify, or the build will fail.

Deployment Process Timeline

Step Duration What Happens
1. Build Start ~10 seconds Netlify clones repository and installs dependencies
2. Creator Scripts ~30-60 seconds Fetch content from Strapi APIs
3. Nuxt Generate ~2-3 minutes Generate 175+ static HTML pages
4. Deploy ~10 seconds Upload files to CDN
Total ~3-5 minutes From push to live

Monitoring Deployments

You can monitor deployment status:

  • Netlify Dashboard: https://app.netlify.com/sites/icjia-infonet/deploys
  • Build Badge: Check the README.md for deployment status badge
  • Build Logs: View detailed logs in Netlify dashboard

Rollback Procedures

If a deployment causes issues, you can rollback:

  1. Go to Netlify dashboard → Deploys
  2. Find a previous successful deployment
  3. Click "Publish deploy" to restore that version
Instant Rollback: Netlify keeps all previous deployments, so you can rollback to any previous version instantly without rebuilding.
