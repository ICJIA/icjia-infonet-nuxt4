### Skip-link defect found + fixed during audit

**Symptom:** First Lighthouse audit on `/` returned **A11y = 97**, flagging `skip-link: No skip link target` on `body > nav.skip-links > a.skip-link`.

**Root cause:** The Phase 1 BaseLayout included a second skip link `<a href="#main-navigation">Skip to navigation</a>`, but `#main-navigation` doesn't exist in the DOM yet — that ID lands in Phase 2 when `AppNav.astro` ships.

**Fix:** commit `9915671` — removed the second skip-link from `BaseLayout.astro` and left a comment noting Phase 2 reintroduces it when the nav lands.

**Result after fix:** Lighthouse a11y = 100 on `/`. axe-core AA = 0 violations.

**Lesson for v6 checklist:** Skip-link targets must exist in the DOM at the phase they ship. Don't write forward-looking skip links. Update `docs/astro-conversion-checklist-v6.2.md` "Phase 1 / BaseLayout" section to note this.
